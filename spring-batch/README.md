# 스프링 배치 조사 문서 안내

## 이 폴더는 무엇을 담고 있나

이 폴더는 `Spring Batch`를 처음 접하는 사람도 빠르게 이해할 수 있게 정리한 조사 문서 묶음이다. 개념 설명부터 어떤 상황에 잘 맞는지, 어디까지 써야 하는지, 시작할 때 무엇을 준비해야 하는지 순서대로 읽을 수 있게 구성했다.

## 추천 읽기 순서

### 1. 가장 먼저 보기

- `01_overview.md`
- 목적: 비전문가도 이해할 수 있는 한눈 요약
- 다루는 내용: 스프링 배치가 무엇인지, 왜 쓰는지, 대표 사용 사례, 한계

### 2. 핵심 구조를 이해하고 싶다면

- `02_core-concepts.md`
- 목적: Job, Step, Reader, Processor, Writer 같은 핵심 개념 이해
- 다루는 내용: 청크 처리, Tasklet, 재시작, skip, retry, 스케줄러와의 관계

### 3. 우리 상황에 맞는지 판단하고 싶다면

- `03_when-to-use.md`
- 목적: 언제 적합하고 언제 과한지 판단
- 다루는 내용: 적합한 사례, 부적합한 사례, 장단점, 판단 체크리스트

### 4. 직접 시작해 보고 싶다면

- `04_getting-started.md`
- 목적: 최소 구성과 시작 순서 파악
- 다루는 내용: 기본 스택, 설계 순서, 간단한 예시 코드, 실무 팁

### 5. 스케줄링과 비교해서 이해하고 싶다면

- `05_batch-vs-scheduler.md`
- 목적: Spring Batch, Quartz, `@Scheduled`의 역할 차이 이해
- 다루는 내용: 책임 구분, 언제 무엇을 써야 하는지, 같이 쓰는 방식

### 6. 면접 대비로 빠르게 정리하고 싶다면

- `06_interview-qa.md`
- 목적: 자주 나오는 질문과 짧은 답변으로 빠르게 복습
- 다루는 내용: 핵심 개념, 재시작, 트랜잭션, 메타데이터, 설계 포인트

### 7. 근거를 확인하려면

- `10_sources.md`
- 목적: 공식 문서와 참고 자료 출처 확인
- 다루는 내용: Spring 공식 레퍼런스, 프로젝트 페이지, 버전 확인 포인트

## 문서 간 관계

```text
01_overview
  -> 02_core-concepts
  -> 03_when-to-use
  -> 04_getting-started
  -> 05_batch-vs-scheduler
  -> 06_interview-qa
  -> 10_sources
```

- `01_overview.md`는 전체 지형을 잡아 주는 입문 문서다.
- `02_core-concepts.md`는 실제로 스프링 배치가 어떤 단위로 움직이는지 설명한다.
- `03_when-to-use.md`는 도입 판단용 문서다.
- `04_getting-started.md`는 가장 작은 시작점을 제시한다.
- `05_batch-vs-scheduler.md`는 스케줄러와의 역할 차이를 비교한다.
- `06_interview-qa.md`는 면접 대비용 압축 정리 문서다.
- `10_sources.md`는 위 문서들의 근거를 모아 둔 문서다.

## 빠른 선택 가이드

- `스프링 배치가 뭔지`만 알고 싶으면 `01_overview.md`
- `실무 개념`이 궁금하면 `02_core-concepts.md`
- `도입 여부 판단`이 목적이면 `03_when-to-use.md`
- `바로 만들어 보기`가 목적이면 `04_getting-started.md`
- `Quartz나 @Scheduled와 차이`가 궁금하면 `05_batch-vs-scheduler.md`
- `면접 대비`가 목적이면 `06_interview-qa.md`
- `근거 검증`이 필요하면 `10_sources.md`

## 한 줄 요약

이 폴더는 `입문 이해 -> 구조 파악 -> 도입 판단 -> 시작 가이드 -> 비교 이해 -> 면접 대비 -> 출처 검증` 순으로 읽도록 설계한 스프링 배치 조사 묶음이다.
