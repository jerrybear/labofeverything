# 스프링 배치 vs Quartz vs @Scheduled

## 먼저 한 줄로 구분하면

- `Spring Batch`: 대량 데이터를 안전하게 처리하는 도구
- `Quartz`: 언제 실행할지 정교하게 관리하는 스케줄러
- `@Scheduled`: 스프링 안에서 간단히 주기 실행시키는 방법

헷갈리기 쉬운데, 이 셋은 `서로 대체재`라기보다 `역할이 다른 도구`다.

## 비전문가 기준 비유

- `Spring Batch`: 공장 생산 라인
- `Quartz`: 정교한 일정 관리자
- `@Scheduled`: 간단한 알람 시계

즉, `언제 시작할지`와 `시작한 뒤 어떻게 안정적으로 처리할지`는 다른 문제다.

## 핵심 차이

| 항목 | Spring Batch | Quartz | `@Scheduled` |
|---|---|---|---|
| 주 역할 | 대량 데이터 처리 | 정교한 실행 스케줄링 | 간단한 주기 실행 |
| 강점 | 재시작, skip, retry, 청크 처리 | 영속 스케줄, 클러스터링, 풍부한 트리거 | 설정이 매우 간단 |
| 적합한 일 | 정산, 파일 적재, 대량 동기화 | 특정 시각 실행, 다중 서버 스케줄 제어 | 간단한 배치성 작업, 정리 작업 |
| 한계 | 스케줄러 자체는 아님 | 데이터 처리 프레임워크는 아님 | 영속성, 고급 제어, 다중 인스턴스 제어가 약함 |

## Spring Batch는 무엇을 잘하나

- 수만 건, 수십만 건 이상의 데이터를 단계적으로 처리
- 중간 실패 후 재시작
- 일부 불량 데이터 skip
- 일시적 장애 retry
- 읽기, 가공, 저장을 구조화

즉, `실제 처리 본체`에 강하다.

## Quartz는 무엇을 잘하나

- 복잡한 일정 표현
- 서버 재시작 이후에도 스케줄 유지
- 여러 서버에서 중복 실행 방지용 운영 구조
- 특정 잡 일시중지, 재개, 수동 트리거 같은 제어

즉, `운영 가능한 스케줄 관리`에 강하다.

## @Scheduled는 무엇을 잘하나

- 코드 한두 줄로 주기 실행 시작 가능
- 작은 내부 작업 자동화에 적합
- 별도 스케줄링 시스템 없이 빠르게 붙이기 좋음

예:

- 매일 캐시 정리
- 10분마다 상태 점검
- 임시 통계 갱신

즉, `간단한 예약 실행`에 강하다.

## 가장 흔한 오해

### 오해 1. Quartz가 Spring Batch를 대체한다

아니다.

- Quartz는 `언제 실행할지`
- Spring Batch는 `실행된 후 어떻게 처리할지`

를 담당한다.

### 오해 2. @Scheduled면 모든 스케줄링이 충분하다

작은 서비스에서는 그럴 수 있지만, 아래가 중요해지면 부족할 수 있다.

- 서버 재시작 후 스케줄 보존
- 다중 서버 환경
- 운영자가 잡을 일시중지하거나 다시 실행하는 기능
- 복잡한 트리거 정책

### 오해 3. Spring Batch가 있으면 스케줄러가 필요 없다

아니다.

스프링 배치는 배치 처리를 잘할 뿐, `언제 돌릴지`를 전문적으로 관리하는 도구는 아니다.

## 언제 무엇을 쓰면 되나

### `@Scheduled`만으로 충분한 경우

- 단일 서버
- 단순 주기 실행
- 실패 이력과 고급 제어가 크게 중요하지 않음

예:

- 새벽마다 오래된 임시 파일 정리
- 5분마다 내부 캐시 갱신

### Quartz가 더 적합한 경우

- 여러 서버에서 하나만 실행돼야 함
- 운영 중 스케줄 변경, 중지, 재개가 필요함
- 애플리케이션 재시작 후에도 스케줄 상태를 보존해야 함

예:

- 여러 인스턴스에서 돌아가는 정기 결산 시작 제어
- 특정 시점 예약 발송, 예약 실행 관리

### Spring Batch가 필요한 경우

- 데이터 처리량이 큼
- 실패 후 이어서 처리해야 함
- 읽기, 가공, 저장이 명확히 분리됨
- skip/retry/트랜잭션/처리 이력이 중요함

예:

- 대량 CSV 적재
- 야간 정산
- 회원 상태 일괄 전환
- 대규모 마이그레이션

## 같이 쓰는 가장 흔한 구조

실무에서는 아래 조합이 자주 나온다.

```text
Quartz or cron or @Scheduled
  -> Spring Batch Job 실행
  -> 대량 데이터 처리
```

즉:

- `스케줄러`가 시작 버튼을 누르고
- `스프링 배치`가 실제 일을 한다

## 빠른 선택 기준

- `간단한 주기 작업`이면 `@Scheduled`
- `복잡한 운영 스케줄링`이면 `Quartz`
- `대량 데이터 처리와 실패 복구`가 핵심이면 `Spring Batch`
- `정해진 시간에 대량 배치`를 돌려야 하면 `Quartz + Spring Batch`

## 면접이나 실무에서 말하기 좋은 한 문장

`@Scheduled`와 `Quartz`는 주로 실행 시점을 관리하고, `Spring Batch`는 실행된 작업을 안정적으로 처리하는 데 집중한다.
