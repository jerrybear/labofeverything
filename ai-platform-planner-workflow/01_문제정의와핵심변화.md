# 문제 정의와 핵심 변화

## 목적

AI 시대에 플랫폼 기획자의 효율이 무엇을 의미하는지 정의하고, 기존 방식 대비 어떤 역할 변화가 발생하는지 정리한다.

## 핵심 문제 정의

플랫폼 관련 수요와 문제를 수집해 사업과 조직 전체에 가장 큰 레버리지를 만드는 투자 결정을 빠르게 내리도록 돕는 것이 플랫폼 기획자의 핵심 job-to-be-done이다.

## 왜 지금 이 주제가 중요한가

- 플랫폼 조직은 요청이 많고 이해관계자가 넓어 문서, 회의, 조율 비용이 크다.
- AI는 이 중 반복적이고 텍스트 중심인 업무를 크게 압축할 수 있다.
- 반면 플랫폼 투자는 범위가 넓고 되돌리기 어려워, 사람의 판단 책임은 더 중요해진다.

## 핵심 변화

1. 요청을 받는 방식보다 신호를 구조화하는 방식이 중요해진다.
2. 기획 문서를 처음부터 수작업으로 쓰기보다 AI로 초안을 만들고 사람이 구조와 판단을 보정한다.
3. 회의, VOC, 운영 이슈, 데이터 분석 결과를 AI로 빠르게 압축해 우선순위 회의의 입력 품질을 높인다.
4. 기획자는 산출물 생산자에서 의사결정 설계자이자 검증 책임자로 이동한다.

## 효율의 정의

이 연구에서 효율은 문서를 더 많이 생산하는 것이 아니라, 더 나은 결정을 더 짧은 루프로 만들고 그 결정을 재사용 가능한 방식으로 축적하는 것을 의미한다.

## 효율 측정 관점

- 요청이 들어온 뒤 1차 분류까지 걸리는 시간이 줄어드는가?
- 회의가 끝난 뒤 실제 결정 로그가 확정되기까지 시간이 줄어드는가?
- 초안에서 승인 가능한 메모로 가는 반복 횟수가 줄어드는가?
- 이미 내린 결정을 다시 여는 비율이 줄어드는가?
- 템플릿과 결정 로그가 다음 업무에 재사용되는 비율이 높아지는가?

## 역할 재정의

### AI가 크게 줄여주는 일

- 대량 문서 요약
- 중복 요청 정리
- 비교표와 초안 작성
- 회의 요약과 액션 아이템 정리
- 인터뷰/피드백의 패턴 추출

### 사람에게 더 중요해지는 일

- 문제를 어떻게 정의할지 결정
- 조직 전체 레버리지가 큰 안건을 고르는 일
- 팀 간 이해관계 충돌을 조정하는 일
- 실패했을 때 책임과 후속 조치를 설계하는 일

## 공통 운영 루프

1. VOC, 요청, 데이터, 장애, 전략 변화 같은 신호를 수집한다.
2. AI로 요약하고 군집화해 패턴을 드러낸다.
3. 문제 정의와 가설을 정리한다.
4. 옵션을 비교하고 영향도, 비용, 리스크, 가역성을 본다.
5. 사람이 우선순위를 결정한다.
6. 실행 문서와 커뮤니케이션 초안을 만든다.
7. 릴리즈 후 학습을 회수해 다음 결정에 반영한다.

## 이 문서의 시사점

- 플랫폼 기획자의 생산성 향상은 개인 문서 작성 속도보다 팀의 입력 품질 향상과 의사결정 구조화에서 더 크게 발생한다.
- 따라서 AI 도입 논의도 툴 선택보다 운영 모델 설계가 먼저다.
