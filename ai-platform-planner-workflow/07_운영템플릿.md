# 운영 템플릿

## 목적

플랫폼 기획자가 AI를 활용해 주간, 월간, 분기 운영을 바로 실행할 수 있도록 회의와 문서 템플릿을 제공한다.

## 1. 주간 입력 정리 템플릿

```text
[주간 입력 정리]

- 기간:
- 이 문서가 돕는 결정:
- 주요 입력 출처:
- 출처 신뢰 수준:
- 기준선:
- 목표:
- 책임자:
- 검토 시점:
- 반복 요청 패턴:
- 새로 발생한 운영 이슈:
- 이상 징후 지표:
- 이번 주 추가 확인이 필요한 항목:
- AI 1차 요약 결과:
- 사람이 수정한 최종 해석:
```

## 2. 주간 우선순위 회의 템플릿

```text
[주간 우선순위 회의]

- 회의 목적:
- 참석자:
- 검토 안건:
- 안건별 근거 출처:
- 안건별 영향도:
- 안건별 확신도:
- 안건별 비용:
- 안건별 가역성:
- 안건별 증거 수준: 강함 / 중간 / 약함
- 최종 결정:
- 보류 사유:
- 후속 액션:
```

## 3. 월간 플랫폼 투자 검토 템플릿

```text
[월간 플랫폼 투자 검토]

- 이번 달 핵심 병목:
- 공통 플랫폼 투자 후보:
- 각 후보의 출처와 근거 수준:
- 각 후보의 주장 상태:
- 각 후보의 기대 효과:
- 조직 전체 레버리지:
- 예상 비용과 제약:
- 리스크:
- 추천 시나리오:
- 이번 달 결정:
```

## 4. 분기 로드맵 시나리오 템플릿

```text
[분기 로드맵 시나리오]

- 시나리오 이름:
- 전제 조건:
- 핵심 목표:
- 포함되는 플랫폼 투자:
- 제외되는 요청:
- 기대 효과:
- 주요 리스크:
- 되돌릴 수 있는 지점:
- 의사결정 필요 사항:
```

## 5. 결정 로그 템플릿

```text
[결정 로그]

- 결정 제목:
- 결정 일자:
- 결정자:
- 배경:
- 고려한 옵션:
- 주요 근거와 출처:
- 근거 수준: 강함 / 중간 / 약함
- 최종 선택:
- 선택 이유:
- 반대 의견 또는 우려:
- 추후 재검토 시점:
```

## 6. 운영 리듬 예시

- 월요일: 주간 입력 정리와 AI 1차 분류
- 수요일: 우선순위 회의와 결정 로그 작성
- 월말: 플랫폼 투자 검토 문서 업데이트
- 분기말: 로드맵 시나리오 비교와 다음 분기 핵심 선택 정리

## 프롬프트 연결 가이드

- 주간 입력 정리 템플릿 작성 전: `06_AI프롬프트모음.md`의 1번 프롬프트 사용
- 문제 정의 초안이 필요할 때: `06_AI프롬프트모음.md`의 2번 프롬프트 사용
- 국내 사례를 회의 안건에 넣기 전: `06_AI프롬프트모음.md`의 2-1번 프롬프트 사용
- 우선순위 회의 메모 준비 시: `06_AI프롬프트모음.md`의 4번 프롬프트 사용
- 이해관계자 메시지 조정 시: `06_AI프롬프트모음.md`의 5번 프롬프트 사용
- 회고 작성 시: `06_AI프롬프트모음.md`의 6번 프롬프트 사용

## 템플릿 사용 원칙

- 모든 템플릿은 "이 문서가 어떤 결정을 돕는가"를 먼저 쓰고 시작한다.
- AI가 채운 초안과 사람이 확정한 내용을 구분해 기록한다.
- 템플릿 항목이 반복적으로 비면 삭제하지 말고, 왜 비는지 먼저 확인한다.
- 실제 운영 흐름이 바뀌면 `00_개요.md`의 문서 정합성 체크도 함께 업데이트한다.
- 조사 결과를 회의나 투자 검토에 넣을 때는 출처와 증거 수준을 반드시 같이 적는다.
- 기준선이 없으면 "측정 시작 필요"로 남기고, 마치 전후 비교가 있었던 것처럼 쓰지 않는다.
