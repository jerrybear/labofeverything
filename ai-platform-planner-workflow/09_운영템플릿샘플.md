# 운영 템플릿 샘플

## 목적

`07_운영템플릿.md`를 실제로 어떻게 채울 수 있는지 보여주는 예시를 만든다.

## 샘플 1. 주간 입력 정리와 우선순위 회의

### 시나리오

- 조직: 국내 중견 핀테크사의 공통 플랫폼 팀
- 배경: 여러 서비스팀이 공통으로 쓰는 운영 지식 검색과 장애 대응 문서 체계가 흩어져 있어, 반복 문의와 회의 시간이 길어지는 상황
- 이번 주 목표: "내부 운영 지식 검색 고도화"를 우선 투자할지 판단하기 위한 입력 정리

### 주간 입력 정리 샘플

```text
[주간 입력 정리]

- 기간: 2026-03-09 ~ 2026-03-13
- 이 문서가 돕는 결정: 내부 운영 지식 검색 고도화 안건을 다음 주 우선순위 회의에 올릴지 여부
- 주요 입력 출처: VOC 23건, 운영팀 문의 로그, 장애 회고 2건, CS 전달 메모
- 출처 신뢰 수준: medium
- 기준선: 운영 관련 반복 문의 답변에 평균 18분 소요
- 목표: 반복 문의 답변 시간을 10분 이하로 낮출 수 있는지 검토
- 책임자: 플랫폼 기획 리드
- 검토 시점: 주간 우선순위 회의
- 반복 요청 패턴: 배치 실패 대응 절차, 알림 설정 위치, 운영 점검 체크리스트 질문 반복
- 새로 발생한 운영 이슈: 신규 팀 합류 후 문서 위치를 찾지 못해 점검 누락 1건 발생
- 이상 징후 지표: 운영 문의량 주간 17% 증가
- 이번 주 추가 확인이 필요한 항목: 어떤 문의가 검색 부재 때문이고 어떤 문의가 문서 품질 문제인지 구분 필요
- AI 1차 요약 결과: 전체 문의 중 약 60%가 문서 존재 여부보다 탐색 비용 문제로 보임
- 사람이 수정한 최종 해석: 검색 기능 고도화보다 문서 표준화와 메타데이터 정리가 함께 필요
```

### 주간 우선순위 회의 샘플

```text
[주간 우선순위 회의]

- 회의 목적: 내부 운영 지식 검색 고도화 안건의 우선 투자 여부 판단
- 참석자: 플랫폼 기획 리드, 운영팀 리드, SRE, CS 운영 담당
- 검토 안건: 운영 지식 검색 고도화 / 장애 체크리스트 표준화 / 공통 알림 설정 UI 개선
- 안건별 근거 출처:
  - 운영 지식 검색 고도화: 문의 로그, 회고 문서, 운영팀 인터뷰
  - 장애 체크리스트 표준화: 회고 문서 2건
  - 공통 알림 설정 UI 개선: VOC 6건
- 안건별 영향도:
  - 검색 고도화: 높음
  - 체크리스트 표준화: 중간
  - UI 개선: 중간
- 안건별 확신도:
  - 검색 고도화: 중간
  - 체크리스트 표준화: 높음
  - UI 개선: 중간
- 안건별 비용:
  - 검색 고도화: 중간
  - 체크리스트 표준화: 낮음
  - UI 개선: 중간
- 안건별 가역성:
  - 검색 고도화: 높음
  - 체크리스트 표준화: 높음
  - UI 개선: 중간
- 안건별 증거 수준: 검색 고도화=medium, 체크리스트 표준화=medium, UI 개선=weak
- 최종 결정: 체크리스트 표준화를 즉시 진행하고, 검색 고도화는 2주 파일럿으로 시작
- 보류 사유: UI 개선은 직접 효과 대비 근거가 약함
- 후속 액션: 검색 파일럿 범위 정의, 문서 메타데이터 표준안 작성
```

## 샘플 2. 월간 투자 검토와 결정 로그

### 시나리오

- 조직: 국내 플랫폼 조직 내 데이터/운영 공통 기능 팀
- 배경: 월간 보고서 초안 작성과 지표 해석에 시간이 오래 걸려, AI 기반 요약/보고 초안 자동화를 투자 후보로 검토 중
- 이번 달 목표: "운영 보고 초안 자동화"를 파일럿 투자 대상으로 올릴지 결정

### 월간 플랫폼 투자 검토 샘플

```text
[월간 플랫폼 투자 검토]

- 이번 달 핵심 병목: 월간 운영 보고서 초안 작성과 수치 해석에 리드타임이 길다.
- 공통 플랫폼 투자 후보: 운영 보고 초안 자동화, 내부 검색 고도화, 승인 로그 일원화
- 각 후보의 출처와 근거 수준:
  - 운영 보고 초안 자동화: 내부 인터뷰, 문서 작성 리드타임 측정 시작 / medium
  - 내부 검색 고도화: 문의 로그 분석 / medium
  - 승인 로그 일원화: 회고 문서와 승인 누락 사례 / medium
- 각 후보의 주장 상태:
  - 운영 보고 초안 자동화: inference
  - 내부 검색 고도화: inference
  - 승인 로그 일원화: observed
- 각 후보의 기대 효과:
  - 운영 보고 초안 자동화: 초안 작성 시간 단축, 반복 설명 감소
  - 내부 검색 고도화: 문의 대응 시간 단축
  - 승인 로그 일원화: 승인 누락과 재논의 감소
- 조직 전체 레버리지: 승인 로그 일원화 > 내부 검색 고도화 > 운영 보고 초안 자동화
- 예상 비용과 제약: 자동화 안건은 보안 검토 필요, 검색 고도화는 메타데이터 정비 비용 존재
- 리스크: 요약 품질이 낮으면 잘못된 보고를 빠르게 만들 수 있음
- 추천 시나리오: 승인 로그 일원화를 즉시 추진하고, 보고 초안 자동화는 제한된 범위 파일럿으로 시작
- 이번 달 결정: 승인 로그 일원화 우선, 보고 초안 자동화는 1개 팀 파일럿 승인
```

### 결정 로그 샘플

```text
[결정 로그]

- 결정 제목: 운영 보고 초안 자동화 파일럿 진행 여부
- 결정 일자: 2026-03-31
- 결정자: 플랫폼 기획 리드
- 배경: 월간 보고 초안 작성 시간이 길고 보고 품질 편차가 큼
- 고려한 옵션: 즉시 전면 도입 / 1개 팀 파일럿 / 보류
- 주요 근거와 출처: 내부 문서 작성 리드타임 측정, 팀 인터뷰, 운영 리스크 검토 메모
- 근거 수준: 중간
- 최종 선택: 1개 팀 파일럿
- 선택 이유: 기대 효과는 있으나 기준선 데이터가 아직 충분하지 않음
- 반대 의견 또는 우려: 보안 민감 문서가 섞일 수 있어 입력 통제가 필요하다는 의견
- 추후 재검토 시점: 파일럿 4주 후
```

## 샘플 사용 팁

- 실제 팀에서 쓸 때는 `기준선`과 `검토 시점`을 반드시 채운다.
- 샘플 문구는 그대로 복사하기보다, 각 팀의 병목과 승인 구조에 맞게 바꾼다.
- 파일럿 안건은 `근거 수준이 중간 이하일 때 어떻게 시작하는가`를 보여주는 예시로 보는 것이 좋다.

## 샘플 3. 규제형 조직의 주간 운영 예시

### 시나리오

- 조직: 국내 금융사 플랫폼 기획 조직
- 배경: 규제 변경과 내부 통제 요구가 많아, 보고서 초안과 승인 로그 관리가 병목인 상황
- 이번 주 목표: 규제 검토 자동화 보조 도구를 파일럿 대상에 올릴지 검토

### 주간 입력 정리 샘플

```text
[주간 입력 정리]

- 기간: 2026-04-06 ~ 2026-04-10
- 이 문서가 돕는 결정: 규제 검토 보조 도구 파일럿 착수 여부
- 주요 입력 출처: 준법감시 체크리스트, 내부 승인 로그, 규제 변경 메모, 운영 이슈 4건
- 출처 신뢰 수준: strong
- 기준선: 규제 변경 반영 초안 작성에 평균 2영업일 소요
- 목표: 동일 업무를 1영업일 이내 초안 작성 수준으로 줄일 수 있는지 검토
- 책임자: 플랫폼 기획 리드, 준법감시 협업 담당
- 검토 시점: 주간 심의 회의
- 반복 요청 패턴: 동일 규제 문구 해석 요청 반복, 승인 경로 누락, 검토 이력 추적 어려움
- 새로 발생한 운영 이슈: 승인 누락으로 검토 회차가 1회 추가됨
- 이상 징후 지표: 규제 변경 반영 backlog 12건 유지
- 이번 주 추가 확인이 필요한 항목: 외부 AI 입력 가능 범위와 내부망 사용 가능 범위 구분
- AI 1차 요약 결과: 반복 검토 중 40%는 규정 검색과 비교표 작성 중심
- 사람이 수정한 최종 해석: 자동화보다 승인 로그 일원화와 규정 검색 보조를 먼저 파일럿해야 함
```

## 샘플 4. 성장형 조직의 월간 운영 예시

### 시나리오

- 조직: 빠르게 성장하는 커머스 스타트업의 공통 플랫폼 팀
- 배경: 조직이 커지면서 같은 운영 질문이 반복되고, 배포/공지/가이드 문서가 분산돼 있어 기능 출시보다 조율 비용이 커진 상황
- 이번 달 목표: 내부 검색 + 공통 안내봇 투자 우선순위 판단

### 월간 플랫폼 투자 검토 샘플

```text
[월간 플랫폼 투자 검토]

- 이번 달 핵심 병목: 신규 팀 합류 증가로 공통 운영 질문이 반복되고, 공지와 가이드 재설명 비용이 커짐
- 공통 플랫폼 투자 후보: 내부 검색 + 안내봇, 배포 체크리스트 자동화, FAQ 문서 구조 개편
- 각 후보의 출처와 근거 수준:
  - 내부 검색 + 안내봇: 문의 로그, 온보딩 인터뷰 / medium
  - 배포 체크리스트 자동화: 배포 사고 회고 / medium
  - FAQ 문서 구조 개편: 팀 인터뷰 / weak
- 각 후보의 주장 상태:
  - 내부 검색 + 안내봇: inference
  - 배포 체크리스트 자동화: observed
  - FAQ 문서 구조 개편: hypothesis
- 각 후보의 기대 효과:
  - 내부 검색 + 안내봇: 반복 설명 감소, 신규 팀 적응 속도 개선
  - 배포 체크리스트 자동화: 배포 누락 감소
  - FAQ 문서 구조 개편: 문서 탐색성 개선
- 조직 전체 레버리지: 내부 검색 + 안내봇 > 배포 체크리스트 자동화 > FAQ 구조 개편
- 예상 비용과 제약: 검색 품질 확보를 위한 메타데이터 정비 필요
- 리스크: 문서 품질이 낮으면 검색 품질도 낮아질 수 있음
- 추천 시나리오: 안내봇 단독 도입보다 검색 구조 정비와 함께 1개 조직 파일럿 진행
- 이번 달 결정: 내부 검색 + 안내봇 파일럿 승인, FAQ 개편은 검색 구조 정비 이후 재검토
```
