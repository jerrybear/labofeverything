# 의사결정 프레임워크

## 목적

AI를 활용하더라도 플랫폼 기획자가 놓치지 말아야 할 의사결정 기준, 인간 승인 경계, 주요 리스크를 정리한다.

## 기본 원칙

- 아티팩트보다 의사결정을 먼저 정의한다.
- AI는 발산과 압축에 강하고, 책임 판단에는 약하다.
- 되돌리기 어려운 결정일수록 사람의 명시적 승인 단계를 둔다.

## 의사결정 질문

1. 지금 해결하려는 문제는 누구의 어떤 손실을 줄이는가?
2. 이 문제가 로컬 최적화가 아니라 공통 플랫폼 투자로 풀릴 이유가 있는가?
3. 지금이 해결 시점인가, 아니면 더 큰 선행 조건이 있는가?
4. 실행 비용 대비 조직 전체 레버리지가 충분한가?
5. 실패했을 때 되돌릴 수 있는가?

## 4축 평가 모델

- 영향도: 사업과 조직 전체에 미치는 효과
- 확신도: 데이터, 인터뷰, 운영 이슈 등 근거의 질
- 비용: 개발 비용, 전환 비용, 운영 복잡도
- 가역성: 잘못된 결정일 때 되돌릴 수 있는 정도

## 4축 평가 예시

```text
[안건 예시] 사내 운영 지식 검색 도구 고도화

- 영향도: 높음
- 확신도: 중간
- 비용: 중간
- 가역성: 높음

해석:
- 여러 팀의 반복 질문과 문서 탐색 시간을 줄일 가능성이 높다.
- 다만 실제 사용률과 답변 품질 데이터가 부족하면 확신도는 높게 주지 않는다.
- 파일럿으로 시작할 수 있으면 가역성은 높다.
```

## 4축 점수 기준

- 영향도: 1=국소적 편의 개선, 3=여러 팀 공통 생산성 개선, 5=조직 전체 레버리지 변화
- 확신도: 1=추정과 의견 중심, 3=일부 데이터와 반복 관찰 존재, 5=명확한 전후 지표와 검증 완료
- 비용: 1=작고 빠른 실행, 3=중간 수준 투자와 조율 필요, 5=큰 개발 비용과 전환 비용 필요
- 가역성: 1=되돌리기 매우 어려움, 3=부분 롤백 가능, 5=파일럿/실험으로 쉽게 중단 가능

## 승인 에스컬레이션 규칙

- 가역성이 낮고 확신도도 낮은 안건은 명시적 인간 승인 없이 진행하지 않는다.
- 보안, 정책, 고객 데이터가 얽힌 안건은 영향도와 무관하게 승인 단계를 둔다.
- 인접 증거만 있는 안건은 파일럿 범위에서만 먼저 실행한다.

## AI 사용 우선순위

### AI에게 먼저 맡길 것

- 대량 문서 요약
- 반복 형식의 초안 작성
- 비교표와 옵션 정리
- 회의 요약과 액션 정리
- 인터뷰와 피드백의 테마 추출

### 사람이 끝까지 잡아야 할 것

- 우선순위 최종 결정
- 조직 간 이해관계 조정
- 아키텍처, 정책, 보안 관련 큰 판단
- 성공 기준 정의
- 실패 책임과 후속 조치 설계

## 대표 리스크

### 속도가 통찰을 가리는 문제

AI는 그럴듯한 문서를 빨리 만들지만, 문제 설정이 틀리면 잘못된 방향으로 더 빨리 달리게 만든다.

### 요약 과정에서 맥락이 사라지는 문제

중요한 예외 케이스, 정치적 맥락, 조직 감정선이 AI 요약에서 빠질 수 있다.

### 산출물 과잉 문제

문서와 옵션은 많아졌는데 실제 결정 품질은 오르지 않을 수 있다.

### 거버넌스 부채

내부 문서, 고객 데이터, 민감한 운영 이슈를 AI에 넣는 과정에서 보안과 정책 리스크가 생길 수 있다.

## 실무용 판단 문장

- "이 문서는 무엇을 설명하는가"보다 "이 문서로 어떤 결정을 내릴 것인가"를 먼저 쓴다.
- "AI가 추천했다"는 근거가 아니라 출발점이다.
- "빠르게 작성"보다 "반복 가능하게 구조화"를 우선한다.
