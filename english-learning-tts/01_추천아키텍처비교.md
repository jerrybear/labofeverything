# 아이용 영어 문장 학습 서비스 추천 아키텍처 비교

## 한눈에 결론

- 가장 균형 좋은 시작은 `Gemini 또는 OpenAI + ElevenLabs` 조합이다.
- 가장 빠른 데모는 `OpenAI 단일 스택`이다.
- 발음 제어와 장기 운영 안정성을 더 중시하면 `Gemini + Google Cloud TTS`가 유력한 대안이다.

## 비교 대상

이번 문서에서는 아래 3가지를 비교한다.

1. 추천안 A: `Gemini 또는 OpenAI + ElevenLabs`
2. 추천안 B: `Gemini + Google Cloud TTS`
3. 빠른안 C: `OpenAI LLM + OpenAI TTS`

## 추천안 A

## `Gemini 또는 OpenAI + ElevenLabs`

### 구조

- 입력 문장 정규화
- LLM이 한국어 설명 JSON 생성
- ElevenLabs가 영어 문장 오디오 생성
- 앱은 한국어 설명 텍스트와 영어 오디오를 함께 표시

### 왜 추천하는가

- 아이가 가장 먼저 느끼는 품질은 음성이다.
- 영어 단문 학습에서는 원어민처럼 자연스럽게 들리는 경험이 중요하다.
- 설명 품질은 LLM 구조화 출력으로 충분히 안정화할 수 있다.

### 장점

- 영어 오디오 만족도가 높을 가능성이 크다.
- 이후 캐릭터 보이스나 감정 톤 확장이 쉽다.
- 설명과 음성을 분리해 각자 최적 벤더를 쓸 수 있다.

### 단점

- 2개 벤더를 관리해야 한다.
- 비용 추적과 장애 대응이 조금 더 복잡하다.
- 발음 예외 사전과 보이스 운영 정책을 따로 가져가야 한다.

### 가장 잘 맞는 경우

- 영어 듣기 경험이 핵심 경쟁력이다.
- 부모가 들어도 오디오 품질 차이를 느끼게 만들고 싶다.
- 단순 도구보다 교육 콘텐츠 경험에 가깝게 만들고 싶다.

## 추천안 B

## `Gemini + Google Cloud TTS`

### 구조

- 입력 문장 정규화
- Gemini가 한국어 설명 JSON 생성
- Google Cloud TTS가 영어 문장 오디오 생성
- 필요 시 custom pronunciation, SSML, speaking rate 사용

### 왜 유력한 대안인가

- 한국어와 영어를 모두 다루기 좋다.
- 발음 제어와 운영 일관성이 좋다.
- 나중에 STT, 음성 평가, 다국어 확장까지 연결하기 쉽다.

### 장점

- 교육용 운영에 필요한 제어 기능이 많다.
- 한 벤더 안에서 통합 관리하기 좋다.
- 장기적으로 커리큘럼 기반 콘텐츠 운영에 잘 맞는다.

### 단점

- 오디오의 감성적 자연스러움은 일부 상황에서 ElevenLabs보다 덜 매력적일 수 있다.
- 초반 설정이 다소 무겁다.

### 가장 잘 맞는 경우

- 서비스 운영 기간이 길다.
- 향후 말하기 평가나 다국어 확장이 계획돼 있다.
- 안정성과 제어를 음성 개성보다 우선한다.

## 빠른안 C

## `OpenAI LLM + OpenAI TTS`

### 구조

- OpenAI가 설명 JSON 생성
- 같은 계열 TTS로 영어 오디오 생성

### 왜 쓸 만한가

- 구현이 가장 빠르다.
- 하나의 인증, 하나의 과금 체계로 시작할 수 있다.
- 초기 사용자 반응을 보기 위한 데모에 적합하다.

### 장점

- 개발 속도가 빠르다.
- 구조가 단순해 디버깅이 쉽다.
- LLM과 TTS 연결 실험이 쉽다.

### 단점

- 장기적으로 영어 TTS를 다른 벤더로 교체하고 싶어질 가능성이 있다.
- 발음 제어 옵션은 Google 계열보다 제한적일 수 있다.

### 가장 잘 맞는 경우

- 이번 주 안에 프로토타입을 보여줘야 한다.
- 실제 제품보다 사용자 반응 확인이 우선이다.
