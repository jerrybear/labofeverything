# 아이용 영어 문장 학습 서비스 MVP PRD

## 제품 한 줄 설명

아이가 쉬운 영어 문장을 입력하면, 원어민처럼 영어 문장을 읽어 주고, 한국어로 뜻과 구성을 짧고 쉽게 설명해 주는 학습 도구를 만든다.

## 문제 정의

초기 영어 학습에서는 아래 문제가 자주 생긴다.

- 짧은 문장을 보고도 어떻게 읽는지 바로 알기 어렵다.
- 뜻은 알아도 문장 구성을 쉽게 설명해 주는 도구가 부족하다.
- 일반 번역기는 학습용 설명이 아니라 결과만 보여준다.
- 아이에게는 설명이 너무 길거나 어렵게 나오는 경우가 많다.

## 목표

MVP의 목표는 아주 단순하다.

- 쉬운 영어 문장 1개를 넣으면 바로 들을 수 있게 한다.
- 아이가 이해할 수준으로 뜻과 구조를 짧게 보여준다.
- 반복 청취가 쉬운 경험을 만든다.

## 비목표

- 자유 대화형 영어 튜터 만들기
- 발음 점수 채점 기능 제공
- 긴 지문 학습 지원
- 학교용 LMS 기능 제공

## 대상 사용자

### 1차 사용자

- 영어 입문 단계의 유아 또는 초등 저학년 아동

### 2차 사용자

- 부모
- 아이 영어 학습을 보조하는 교사 또는 튜터

## 핵심 사용자 시나리오

### 시나리오 1

- 아이 또는 부모가 `I am a boy.`를 입력한다.
- 서비스가 영어 문장을 자연스럽게 읽어 준다.
- 화면에 뜻과 쉬운 설명이 나온다.
- 사용자는 다시 듣기를 누른다.

### 시나리오 2

- 사용자가 `This is my book.`를 입력한다.
- 서비스는 문장을 읽고, `이것은 내 책이야.`라고 뜻을 보여 준다.
- 구성 설명에서 `This is ...` 패턴을 쉽게 알려 준다.

## 핵심 가치 제안

- 번역이 아니라 학습용 설명을 준다.
- 영어 듣기와 뜻 이해를 한 번에 연결한다.
- 아이 수준에 맞는 짧고 쉬운 설명을 제공한다.

## MVP 범위

### 포함

- 영어 문장 입력
- 영어 TTS 재생
- 한국어 뜻 표시
- 한국어 문장 구성 설명
- 아이용 한 줄 보충 설명
- 다시 듣기
- 느린 속도 재생 선택 기능

### 제외

- 실시간 음성 대화
- 학습 계정 시스템
- 교사 대시보드
- 발음 평가
- 진도 관리

## 사용자 경험 원칙

- 입력 후 결과는 빠르게 보여 준다.
- 화면은 영어 문장, 재생 버튼, 한국어 설명 중심으로 단순하게 구성한다.
- 설명은 짧고 쉬워야 한다.
- 어려운 문법 용어는 피한다.

## 기능 요구사항

### 1. 입력

- 사용자는 영어 문장 1개를 입력할 수 있어야 한다.
- 공백이나 대소문자 차이는 자동으로 정리되어야 한다.
- 너무 길거나 어려운 문장은 거절하거나 짧게 다시 써 달라고 안내해야 한다.

### 2. 설명 생성

- 시스템은 한국어 뜻을 생성해야 한다.
- 시스템은 문장 구성을 쉬운 한국어로 설명해야 한다.
- 시스템은 아이용 한 줄 보충 설명을 생성해야 한다.
- 설명은 구조화된 스키마로 생성되어야 한다.

### 3. 음성 생성

- 시스템은 영어 문장을 자연스럽게 읽는 오디오를 생성해야 한다.
- 기본 속도 오디오를 제공해야 한다.
- 선택적으로 느린 속도 오디오도 제공해야 한다.

### 4. 결과 표시

- 영어 문장을 크게 보여 줘야 한다.
- 재생 버튼은 가장 눈에 잘 띄어야 한다.
- 뜻과 구성 설명은 짧은 블록으로 구분해서 보여 줘야 한다.

## 비기능 요구사항

### 응답 속도

- 일반적인 짧은 문장은 체감상 빠르게 결과가 나와야 한다.
- 캐시된 문장은 더 빨리 제공되어야 한다.

### 품질

- 영어 발음은 입문 학습용으로 충분히 자연스러워야 한다.
- 한국어 설명은 아이 수준에서 이해 가능해야 한다.
- 자주 쓰는 샘플 문장 세트에 대해 수동 검수가 가능해야 한다.

### 운영

- 같은 문장 요청은 캐시할 수 있어야 한다.
- TTS provider는 추후 교체 가능해야 한다.
- 설명 모델 버전과 TTS 버전을 기록해야 한다.

## 성공 지표

| 지표 | 정의 | 초기 목표 |
| --- | --- | --- |
| first play rate | 결과 화면에서 첫 재생 버튼을 누른 비율 | 높을수록 좋음 |
| replay rate | 같은 문장을 2회 이상 다시 들은 비율 | 높을수록 좋음 |
| explanation helpful rate | 부모 또는 테스트 사용자가 설명이 이해에 도움이 됐다고 평가한 비율 | 80% 이상 목표 |
| audio satisfaction | 영어 음성이 자연스럽다고 평가한 비율 | 80% 이상 목표 |
| failure rate | 설명 또는 음성 생성 실패 비율 | 낮을수록 좋음 |

## 품질 검수 기준

골든셋 문장 100~200개를 만들어 아래를 점검한다.

- 뜻이 정확한가
- 설명이 쉬운가
- 문법 용어가 과하지 않은가
- 음성이 자연스러운가
- 어색한 문장은 자연스럽다고 잘못 안내하지 않는가

## 주요 리스크

### 1. 설명이 너무 어렵게 나올 수 있음

대응:

- 출력 길이 제한
- 금지 용어 목록
- 구조화 출력 강제

### 2. 음성이 충분히 자연스럽지 않을 수 있음

대응:

- 초기부터 2~3개 TTS 후보를 비교 청취
- 골든셋으로 반복 비교

### 3. 이상한 입력이 들어올 수 있음

대응:

- 난이도 필터
- 길이 제한
- 지원 범위 밖 문장 안내

## 권장 기술 방향

- 설명: Gemini 또는 OpenAI 계열의 구조화 출력
- 음성: ElevenLabs 우선 검토, Google Cloud TTS 대안 유지
- 오케스트레이션: 서버 1개 엔드포인트에서 설명과 TTS 조합

## MVP 출시 기준

아래가 충족되면 MVP 출시 가능 상태로 본다.

- 짧은 영어 문장 입력부터 결과 표시까지 기본 흐름이 안정적으로 동작한다.
- 골든셋에서 설명 품질과 음성 품질이 기준 이상이다.
- 캐시와 에러 응답이 동작한다.
- 같은 문장을 반복 요청해도 품질이 크게 흔들리지 않는다.
