# 개발자 은퇴 이후 커리어 전환 연구

이 폴더는 AI 에이전트 시대에 개발자가 현업에서 밀려나거나 스스로 물러난 뒤, 실제로 무엇을 할 수 있는지 더 차갑고 현실적으로 정리하는 작업 공간이다.

## 현재 구성

- `00_개요.md`: 연구 문제의식, 핵심 주장, 구조 변화를 정리한 개요 문서
- `01_개발자-은퇴-후-무엇을-할까.md`: 은퇴 이후 경로를 칼럼 톤으로 압축해 정리한 메인 노트
- `02_한국-시장-신호.md`: 한국 시장에서 시니어 개발자, 자문, 교육, 마이크로 SaaS 관련 신호를 모은 보강 노트
- `03_왜-자문-시장은-생각보다-작은가.md`: 자문 시장의 착시와 반복 수익이 어려운 이유를 정리한 심화 노트
- `04_왜-강의-시장은-보람보다-노동집약적인가.md`: 교육과 멘토링이 왜 생각보다 시간과 감정노동을 많이 요구하는지 정리한 심화 노트
- `05_왜-마이크로-saas는-자영업이-되는가.md`: 작은 SaaS가 왜 패시브 인컴보다 1인 운영업에 가까워지는지 정리한 심화 노트
- `06_신문-칼럼-버전.md`: 전체 논지를 더 차갑고 압축된 신문 칼럼 톤으로 정리한 문서

## 핵심 관점

- 개발자 은퇴는 종종 휴식의 시작이 아니라 시장가치 재평가의 시작이다.
- AI 에이전트는 코딩을 없애지 않지만 구현의 가격과 위신을 계속 깎아내릴 가능성이 크다.
- 은퇴 후 살아남는 자산은 기술 스택보다 판단력, 신뢰, 고객 접근, 평판, 소개 네트워크다.

## 다음 작업 후보

- 한국 시장에서 시니어 개발자 자문, 프랙셔널 CTO, 고문 역할의 실제 계약 사례 추가 수집
- 강의, 멘토링, 교육 상품화의 실제 단가 구조와 반복 운영 부담 사례 추가 조사
- 마이크로 SaaS의 결제, 세무, 환불, 고객지원 운영 부담 사례 추가 조사
- AI 에이전트 도입 이후 시니어 개발자의 역할 축소와 재정의 사례를 업종별로 분류
