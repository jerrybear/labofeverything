# AGENTS.md

This file defines working rules only for the `너에게-조금-느리게-가고-있어` project.

## Scope

- Apply these rules only inside `너에게-조금-느리게-가고-있어/`.
- Do not assume they apply to other research or writing projects in the workspace.

## Writing Workflow

For each episode, follow this sequence unless the user explicitly asks for a different process.

1. Structure memo
2. `D-XX-초안.md`
3. Draft compression/adjustment pass on `D-XX-초안.md`
4. `F-XX-원고.md`

Do not skip step 3 by default.
The compression/adjustment pass must be treated as a real stage, not just a nominal one.

## Role Of Each File Type

- Structure memo: episode goal, progression, risks, ending direction
- `D-XX-초안.md`: exploratory draft with full scene blocking and narrative testing
- Draft compression/adjustment: remove repetition, reduce explanation, sharpen transitions, strengthen ending
- `F-XX-원고.md`: final reader-facing manuscript

## Draft Compression Checklist

Before converting `D-XX-초안.md` into `F-XX-원고.md`, check for all of the following.

- Repeated explanation of the same idea
- Scene momentum slowing into summary or commentary
- Reuse of the same beat from the previous episode
- Ending line not strong enough to open the next episode
- Length drift that feels out of rhythm with nearby episodes

When reporting the conversion from `D` to `F`, briefly state what was cut, tightened, or strengthened.

## Episode Formatting Rules

- Keep the series title header in each manuscript file
- Keep episode marker format as `## F-XX` or `## D-XX`
- Add one episode subtitle under the episode marker
- Subtitle should name the episode's narrative stage, not just its location

Do not proactively rewrite `F-01-원고.md` or `F-02-원고.md` mid-run unless the user explicitly asks.

They should be revised later with these goals:

- reduce repetitive explanation
- tighten scene transitions

See also: `99-후반-편집-리마인드.md`

## Story Progression Principle

Prefer progression like this:

- not just new place -> new clue
- but clue -> system -> person -> movement -> transfer structure

As the story advances, avoid falling back into repetitive location verification.
Later episodes should keep moving from structure to operator, and from operator to route, timing, and transfer behavior.

## Continuity Principle

When writing a new episode:

- read the latest final manuscript and the latest structure memo first
- preserve the current emotional and investigative handoff from the previous episode
- keep Seoyun's perception mode evolving, not repeating at the same level

## Seoyun Ability Principle

Her ability should not become simple plot convenience.

- Early stage: delayed light, fragmentary seeing
- Middle stage: noticing gaps, surfaces, erased traces
- Later stage: reading habits, rhythms, and movement patterns

Keep the evolution gradual and tied to cost, uncertainty, and partial perception.
